Pre-Tutorial Learning Activities

T10A1. Do a peer code review

  • Refer to the learning outcome [LO-CodeReview]
  • For at least one PR created by you, get a team member to do a code review (on GitHub) before merging the PR.
  • Review at least one PR created by a team member.

T10A2. Reach V0.4

Follow V0.4 Requirements.
  • Remember to collate code and tests written by you (see V0.4 Requirements for more info).
    Your tutor will do a brief review of the collated code to give you some feedback about your code quality.
  • Note that there are special requirements for the V0.4 demo which require advance preparations.

T10A3. Relate Design Patterns to the project

Refer to the learning outcome Apply Design Patterns [LO-DesignPatterns] and the relevant lecture handout. Find which design patterns are used in the AddressBook design. Are they still in your design too?
Find if those patterns can be applied in other places of your design.
